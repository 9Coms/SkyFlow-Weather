# แผนอัปเกรด N.K. Anonymous Blog (Phase 4 - ระบบปราบเกรียน)

รับทราบ จัด 3 ระบบสุดเดือดให้ตามคำขอเพื่อล็อกเป้าและแบนพวกเกรียนให้อยู่หมัด

## สิ่งที่จะเพิ่มเข้าไปใน Frontend ล้วนๆ

### 1. ระบบสายลับ Canvas Fingerprinting (ดึงข้อมูลระดับ Hardware)
- กูจะเขียนฟังก์ชันสร้าง Canvas ซ่อนไว้หลังบ้าน แล้วให้เบราว์เซอร์เรนเดอร์รูปภาพกราฟิกแบบลับๆ 
- ฮาร์ดแวร์และการ์ดจอแต่ละเครื่องจะเรนเดอร์ภาพออกมาต่างกันระดับพิกเซล กูจะเอาภาพนั้นมาสับเป็น **Hash Code (Device ID)**
- เวลามีคนโพสต์ แอดมินจะเห็น `Device ID` โชว์ในหน้า `admin.html` (เช่น `HW-a8f9c2...`) ต่อให้เด็กมันเปิดโหมดไร้ตัวตน หรือเปลี่ยนชื่อ ถ้าเลขนี้ตรงกัน แปลว่าเป็นคนเดียวกันชัวร์

### 2. ระบบช่วยกันสอดส่อง (Community Report)
- ทุกโพสต์ (ที่ไม่ได้ปักหมุดหรือเป็นของแอดมิน) จะมีปุ่ม **"รีพอร์ต 🚩"**
- ระบบจะจำว่าใครกดรีพอร์ตไปแล้วบ้าง (กันกดรัวๆ)
- ถ้าโพสต์ไหนโดนคนรีพอร์ตครบ **3 ครั้ง** โพสต์นั้นจะถูก **ซ่อนทันที** (กลับไปติดสถานะรออนุมัติ) 
- ในหน้าแอดมิน โพสต์นั้นจะมีป้ายเตือนสีแดงว่า **"โดนรีพอร์ต"** ให้แอดมินตัดสินใจว่าจะกดอนุมัติคืนชีพให้ หรือกดลบทิ้งถาวร

### 3. ระบบกรองคำหยาบแบบ Fuzzy Regex (ดักจับคำเลี่ยง)
- กรองคำแบบธรรมดามันอ่อนไป กูจะเขียน Regular Expressions (Regex) ดักจับรูปแบบที่เด็กชอบพิมพ์เลี่ยง
- เช่น `ค.ว.ย`, `ค_ว_ย`, `สัss`, `ส ั ส` หรือการเว้นวรรคแปลกๆ ระบบจะกวาดล้างแล้วเปลี่ยนเป็น `***` ให้หมดแบบอัตโนมัติ

## Proposed Changes

### [MODIFY] `app.js`
- เพิ่มฟังก์ชัน `getCanvasFingerprint()` ตอนกดส่งโพสต์
- เพิ่มฟังก์ชัน `toggleReport(postId)` เพื่อเก็บยอดรีพอร์ต
- อัปเกรดฟังก์ชัน `filterProfanity(text)` ให้เป็น Regex ขั้นสูง

### [MODIFY] `admin.html` & `admin.js`
- เพิ่มการแสดงผล `Device ID` คู่กับ `UserAgent`
- เพิ่มเงื่อนไขการโชว์ป้ายเตือน "โดนรีพอร์ต" ในตารางข้อมูล
